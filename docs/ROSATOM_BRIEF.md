# Бриф для Росатома: воспроизведение MIT Lapp (1989) + ИИ

Операционный runbook: [RUNBOOK.md](RUNBOOK.md) · отчёт: [ROSATOM_RESULTS.md](ROSATOM_RESULTS.md) · метрики: [METRICS_AND_GLOSSARY.md](METRICS_AND_GLOSSARY.md)

## Что уже сделано (текущий benchmark)

- Оцифрована матрица взаимодействий 25×25 (Fig. 6-1, Shearon Harris).
- Современные оптимизаторы (**GA+LS**, SA, Spectral, Louvain) сравнены с **BEA-style** baseline.
- **GA+LS снижает межмодульный разрез (inter-module cut) на ~25–35%** относительно BEA-эвристики 1989 года.
- Исторический **Design Value** Modular SH II ≈ **$51.78M** валидирован из таблиц диссертации (vs Original **$76.45M**, −32.3%).

Это доказательство: матричный этап методологии сегодня можно улучшить готовыми оптимизационными методами.

## Что добавлено в этом этапе (ИИ / следующий слой)

1. **Полное side-by-side Table 9-1** — все 5 компонентов DV для Original vs Modular SH II.
2. **DV-proxy оптимизация** — GA минимизирует не только cut, но и прокси Design Value (MOM из матрицы + суррогаты MAV/CVV/MFV/PPRV).
3. **Экономика Ch.9** — чувствительность капитальной стоимости к сроку строительства и ставке процента; тезисный коридор **26–46%** savings воспроизведён как benchmark ranges.
4. **LLM-навигатор Steps 1–12** — сопровождение инженера по методологии Lapp + оцифрованные constraints (nuclear/safety).
5. **Пайплайн извлечения таблиц из PDF** (`pdftotext` + опциональный LLM).

## Рекомендуемый следующий шаг

Замкнуть оптимизацию на **инженерные ограничения и измеримый DV**, а не добавлять ещё один алгоритм кластеризации «в вакууме».

Практический пакет:

| Приоритет | Работа | Зачем Росатому |
|---|---|---|
| P0 | Воспроизводимый пакет + Table 9-1 | Прозрачная валидация диссертации |
| P1 | DV-proxy vs cut-only | Ответ: улучшает ли ИИ Design Value, а не только cut |
| P2 | LLM-навигатор + constraints | Демо «ИИ как помощник проектировщика» |
| P2 | Экономика Ch.9 | Бизнес-слайд по сроку / ставке / AFUDC |
| P3 | Свой объект (ВВЭР/СМР) | Нужны внутренние P&ID / матрица связей |

## Что мы сознательно не обещаем

- Полный автодизайн АЭС без инженеров.
- Точный пересчёт **PPRV/MAV/CVV** без pipe/layout данных (сейчас — суррогаты).
- Что исторические −32% DV / −26…46% capital cost получены GA+LS — это результат **экспертного redesign SH II** в диссертации.
- Побитовую копию оригинальной программы BEA 1989 (baseline — воспроизведение идеи).

## Одно предложение для письма

> Мы подтвердили, что современные оптимизаторы существенно улучшают матричный этап методологии MIT 1989; следующий шаг с ИИ — связать разбиение модулей с Design Value, инженерными ограничениями и экономикой строительства, а при наличии данных Росатома — перенести benchmark на российский объект.
